import { Tabs, TabItem } from '@astrojs/starlight/components';

Honker's pub/sub is a direct analog of Postgres's `pg_notify` / `LISTEN`: fire-and-forget signals between processes that share a SQLite file. 1-2 ms cross-process wake latency on M-series laptops. No replay, no delivery guarantees. For that, use [Streams](/guides/streams/).

The wire format is the same across bindings: a Python publisher and a Go listener talk to the same channel through the same `.db` file.

## Fire a signal

## Listen

The listener starts from `MAX(id)` at attach time. Historical notifications are not replayed. Honker uses a shared stat-poll thread on the `.db-wal` file with a 1 ms cadence, so every listener in the process wakes on any WAL commit and then filters by channel at the `SELECT` level.

<Tabs syncKey="language">
  <TabItem label="Python">
```python
async for notif in db.listen("orders"):
    print(notif.channel, notif.payload)
```
  </TabItem>
  <TabItem label="Node">
```js
const ev = db.walEvents();
let lastId = db.transaction()
  .query("SELECT COALESCE(MAX(id), 0) AS m FROM _honker_notifications")[0].m;

while (running) {
  const tx = db.transaction();
  const rows = tx.query(
    "SELECT id, channel, payload FROM _honker_notifications WHERE id > ? AND channel = ? ORDER BY id",
    [lastId, 'orders'],
  );
  tx.commit();

  if (rows.length === 0) { await ev.next(); continue; }
  for (const r of rows) {
    console.log(r.channel, JSON.parse(r.payload));
    lastId = r.id;
  }
}
```
  </TabItem>
  <TabItem label="Rust">

## Cross-process

The WAL watcher fires on commits from *any* process. A FastAPI app, a CLI worker, and a cron job can all `notify()` the same channel, and a separate listener receives every signal within a few milliseconds. No coordination, no broker, no extra network hop.

## Pruning

Notifications accumulate in `_honker_notifications` until you prune them. There's no auto-prune because "how much history do you want" is a product decision. Honker exposes two prune modes, available from any binding.

```python
# Python
db.prune_notifications(older_than_s=3600)   # delete rows older than 1 hour
db.prune_notifications(max_keep=10_000)     # keep only the most recent 10k
```

```sql
-- From SQL
SELECT honker_prune_notifications_older_than(3600);
SELECT honker_prune_notifications_keep_latest(10000);
```
